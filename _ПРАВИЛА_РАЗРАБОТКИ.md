## 💎 ЕДИНЫЙ СТАНДАРТ КОДА (V2.0)

### 1️⃣ Архитектура памяти и переменных
**Global Priority:** Программы потребляют 1-10 МБ RAM, современные системы имеют 32+ ГБ. Всегда используем Global для ключевых переменных - это обеспечивает доступность данных в любой точке программы и упрощает отладку.

**Правила:**
- ✅ Используем Global для всех переменных, используемых в разных функциях
- ✅ Уникальность имен: каждая переменная имеет префикс модуля (например, `$g_MySQL_Timeout`, `$g_Redis_Host`)
- ✅ Венгерская нотация обязательна: `$s` (string), `$i` (int), `$a` (array), `$h` (handle), `$b` (boolean), `$f` (float)
- ✅ Все подключения библиотек (#include) размещаются после `#include-once` для предотвращения конфликтов
- ❌ Запрещено: Local для данных, используемых в нескольких функциях

**Пример:**
```autoit
#include-once
#include "Utils/Utils.au3"
#include "json/JSON.au3"

Global $g_MySQL_Host = "localhost"
Global $g_iMySQL_Timeout = 500
Global $g_bMySQL_DebugMode = True
Global $g_aMySQL_Queue = ''

Global $g_sRedis_Host = "127.0.0.1"
Global $g_iRedis_Port = 6379
Global $g_bRedis_Connected = False
```

### 2️⃣ GUI и интерфейс (WebView2 + Native)
**Событийная модель (OnEventMode):** Никаких циклов опроса GUIGetMsg(). Только `Opt("GUIOnEventMode", 1)`.

**Правила:**
- ✅ Все GUI работают через OnEventMode
- ✅ WM-обработчики для сложных контролов: `GUIRegisterMsg(WM_NOTIFY, "_WM_NOTIFY_Handler")`
- ✅ WebView2: каждая страница имеет свой CSS файл
- ✅ Минималистичная красота: анимации/эффекты только по событию (hover/click), в пассивном режиме 0% CPU
- ❌ Запрещено: циклы While/Do с GUIGetMsg(), фоновые анимации

**Пример:**
```autoit
Opt("GUIOnEventMode", 1)
$hGUI = GUICreate("MyApp", 800, 600)
GUISetOnEvent($GUI_EVENT_CLOSE, "_OnExit")
$btnSave = GUICtrlCreateButton("Save", 10, 10, 100, 30)
GUICtrlSetOnEvent($btnSave, "_OnSaveClick")
```

### 3️⃣ Работа с массивами (Anti-ReDim Policy)
**Запрет на ReDim:** Функция ReDim исключена из разработки навсегда из-за проблем с производительностью и стабильностью.

**Правила:**
- ✅ Массивы объявляются глобально: `Global $aArray = ''`
- ✅ Для очистки используем: `$aArray = ''`
- ✅ Проверка перед использованием: `If $aArray <> '' And UBound($aArray, 1) > 0`
- ✅ Двумерные массивы - основной стандарт для данных
- ✅ Одномерные массивы - только для микроопераций в системных библиотеках
- ❌ Запрещено: ReDim, изменение размера массива

**Пример:**
```autoit
Global $g_aMySQL_Results = ''

Func _MySQL_GetData()
    Local $aData[10][3]
    ; ... заполнение данных ...
    $g_aMySQL_Results = $aData
EndFunc

Func _MySQL_ProcessData()
    If $g_aMySQL_Results <> '' And UBound($g_aMySQL_Results, 1) > 0 Then
        ; Обработка данных
    EndIf
EndFunc
```

### 4️⃣ Документирование и шапки (Self-Documenting Code)
**Шапка файла:** В начале каждого .au3 файла обязателен список всех функций с параметрами. Обновляется при каждом изменении.

**Шапка функции:** Над каждой функцией блок описания:
```autoit
; ===============================================================================
; Функция: _MySQL_Select
; Описание: Упрощенный SELECT запрос с детальной диагностикой
; Параметры:
;   $sTable - имя таблицы (обязательный)
;   $sColumns - колонки для выборки (по умолчанию "*")
;   $sWhere - условие WHERE (опционально, без слова WHERE)
;   $sOrderBy - сортировка ORDER BY (опционально)
;   $iLimit - лимит записей (опционально)
;   $iServer - сервер: 1=хостинг, 2=локальный, 3=оба (по умолчанию 2)
;   $bJSON - формат ответа (по умолчанию False)
; Возврат: массив данных или False при ошибке
; Пример: _MySQL_Select("users", "id,name", "status='active'", "name ASC", 10)
; ===============================================================================
```

**Правила:**
- ✅ Каждая функция имеет полное описание
- ✅ Указываются все параметры с типами и значениями по умолчанию
- ✅ Обязателен рабочий пример использования
- ✅ Шапка файла содержит список ВСЕХ функций
- ❌ Запрещено: функции без описания, устаревшие описания

### 5️⃣ Интеллектуальное логирование (Logger V2)
**Все логи через единую функцию** с фильтрацией по типу и месту вывода.

**Инициализация в приложении:**
```autoit
#include "..\..\libs\Utils\Utils.au3"

; Минимальная инициализация (дефолты: debug=True, все логи, консоль+файл, очистка)
_SDK_Utils_Init("MyApp", "Main")

; Полная инициализация
_SDK_Utils_Init("MyApp", "Main", True, 1, 3, True)
; Параметры: AppName, ModuleName, DebugMode, LogFilter, LogTarget, ClearLog
```

**Глобальные переменные (можно менять в любой момент):**
```autoit
$g_sUtils_SDK_AppName = "MyApp"          ; Имя приложения
$g_sUtils_SDK_ModuleName = "Main"        ; Имя модуля
$g_bUtils_SDK_DebugMode = True           ; Включить/выключить логи
$g_iUtils_SDK_LogFilter = 1              ; 1=все, 2=ошибки, 3=успех
$g_iUtils_SDK_LogTarget = 3              ; 1=консоль, 2=файл, 3=оба
```

**Использование:**
```autoit
; Информационное сообщение
_Logger_Write("Приложение запущено", 1)

; Ошибка
_Logger_Write("Не удалось подключиться", 2)

; Успех
_Logger_Write("Данные сохранены", 3)

; Смена модуля на лету
$g_sUtils_SDK_ModuleName = "MySQL"
_Logger_Write("Подключение к MySQL", 1)  ; → logs/MyApp/MyApp_MySQL.log
```

**Структура логов:**
```
logs/
└── AppName/
    ├── AppName_Main.log
    ├── AppName_MySQL.log
    └── AppName_Redis.log
```

**Правила:**
- ✅ Все логи только через `_Logger_Write()`
- ✅ Критические ошибки: `_Logger_Write("Ошибка", 2)`
- ✅ Успешные операции: `_Logger_Write("Успех", 3)`
- ✅ Информация: `_Logger_Write("Инфо", 1)`
- ✅ Логи очищаются при инициализации (если `$bClearLog = True`)
- ✅ Логи накапливаются между запусками (если `$bClearLog = False`)
- ❌ Запрещено: прямой ConsoleWrite, логи без фильтрации

### 6️⃣ Обработка ошибок

**COM объекты (ОБЯЗАТЕЛЬНО при использовании ObjCreate):**
Если в программе используется хотя бы один COM объект, обязательна установка глобального обработчика COM ошибок.

```autoit
; В начале программы (после #include)
Global $g_oCOMError = ObjEvent("AutoIt.Error", "_COM_ErrorHandler")
OnAutoItExitRegister("_COM_Exit")

; Обработчик COM ошибок (НЕ прерывает выполнение)
Func _COM_ErrorHandler()
    Local $sError = "COM Error: " & @error & " | ScriptLine: " & $g_oCOMError.scriptline
    _Logger_Write($sError, 2)
    ; НЕ используем SetError() - это прервёт выполнение
    ; Просто логируем и возвращаемся
    Return
EndFunc

; Очистка COM объектов при выходе
Func _COM_Exit()
    ; Освобождаем COM объекты
    $g_oMyObject = 0
    $g_oCOMError = 0
EndFunc
```

**Обычные функции:**
- ✅ Всегда использовать `SetError()` для возврата ошибок из функций
- ✅ Проверять `@error` сразу после вызова критичных функций
- ✅ Критические ошибки логировать через `_Logger_Write($sError, 2)`
- ✅ Возвращать False при ошибке, True при успехе (для операций без данных)

**Пример:**
```autoit
Func _MySQL_Connect()
    Local $hConn = _SomeConnection()
    If @error Then
        _Logger_Write("Ошибка подключения к MySQL: " & @error, 2)
        Return SetError(1, 0, False)
    EndIf
    Return $hConn
EndFunc
```

### 8️⃣ Неблокирующие циклы и runtime-расширяемость
**Философия:** Программы SDK - это модульные системы реального времени. Множество независимых модулей работают параллельно без блокировки друг друга.

**Правила:**
- ✅ Основной цикл всегда `While 1 ... Sleep(1) ... WEnd`
- ✅ Все задержки через `TimerDiff()` с возвратом в основной цикл
- ✅ Каждый модуль имеет свой таймер и интервал выполнения
- ✅ Модули регистрируются динамически через `_RegisterModule()` для runtime-расширяемости
- ✅ Функции модулей вызываются через `Call()` для гибкости
- ✅ For циклы допустимы только для быстрых операций (<100мс)
- ✅ Ожидание событий через таймеры с таймаутом, без блокировки
- ❌ Запрещено: блокирующие ожидания While, счётчики вместо таймеров, вложенные циклы While

**Базовый шаблон приложения:**
```autoit
; Событийная модель
Opt("GUIOnEventMode", 1)

; Основной цикл (одинаков для всех приложений)
While 1
    ; Обработка модулей через таймеры
    _ProcessModules()
    Sleep(1)  ; Минимальная задержка, освобождает CPU
WEnd

; Таймеры вместо счётчиков
Global $g_hMyTimer = TimerInit()
Global $g_iMyInterval = 1000  ; 1 секунда

Func _MyTask()
    If TimerDiff($g_hMyTimer) >= $g_iMyInterval Then
        _DoSomething()
        $g_hMyTimer = TimerInit()
    EndIf
EndFunc

; Ожидание без блокировки
Global $g_hWaitTimer = 0
Global $g_iWaitTimeout = 5000

Func _CheckEvent()
    If $g_bEventReady Then
        _ProcessEvent()
        $g_hWaitTimer = 0
        Return True
    EndIf
    
    If TimerDiff($g_hWaitTimer) >= $g_iWaitTimeout Then
        _Logger_Write("Таймаут события", 2)
        $g_hWaitTimer = 0
        Return False
    EndIf
    Return False
EndFunc

; Runtime-расширяемость (динамическая регистрация модулей)
Global $g_aModules = ''

Func _RegisterModule($sName, $funcCallback, $iInterval)
    ; Добавление модуля в массив без ReDim
    ; Модуль: [Имя, Функция, Таймер, Интервал]
EndFunc

Func _ProcessModules()
    If $g_aModules <> '' Then
        For $i = 0 To UBound($g_aModules, 1) - 1
            If TimerDiff($g_aModules[$i][2]) >= $g_aModules[$i][3] Then
                Call($g_aModules[$i][1])  ; Вызов функции модуля
                $g_aModules[$i][2] = TimerInit()
            EndIf
        Next
    EndIf
EndFunc

; Регистрация модулей в runtime
_RegisterModule("Redis", "_Redis_Process", 10)      ; Каждые 10мс
_RegisterModule("MySQL", "_MySQL_Sync", 5000)       ; Каждые 5 сек
_RegisterModule("Monitor", "_System_Check", 1000)   ; Каждую секунду
```

### 9️⃣ Безопасность при работе с функциями
**Проблема дубликатов:** В больших проектах могут быть функции с одинаковыми именами в разных файлах.

**Правила:**
- ✅ При использовании MCP функций `read_function`, `write_function`, `delete_function` всегда указывай `file_path` если есть риск дубликатов
- ✅ MCP автоматически проверяет дубликаты и предупреждает о необходимости указать `file_path`
- ✅ `write_function` автоматически добавляет `EndFunc` если отсутствует (любой регистр)
- ✅ Если дубликаты в одном файле - MCP вернет номера строк для ручного разбора
- ❌ Запрещено: изменение функций без указания `file_path` при наличии дубликатов

**Пример:**
```autoit
; Безопасно - указан file_path
_MCP_Write_Function("_Test", $sNewCode, "apps/MyApp/MyApp.au3")

; Опасно - если _Test есть в нескольких файлах, MCP вернет предупреждение
_MCP_Write_Function("_Test", $sNewCode)
```

### Правила модулей:
1. **Один модуль = одна папка** с собственным README.md
2. **README_SDK.md** в корне - карта навигации для AI (таблица модулей + пути к детальным README)
3. **Префиксы обязательны:**
   - Функции: `_ModuleName_FunctionName()` (например, `_MySQL_Select()`, `_Redis_Get()`)
   - Переменные: `$g_ModuleName_VarName` (например, `$g_MySQL_Host`, `$g_Redis_Host`, `$g_Utils_SDK_AppName`)
4. **Зависимости:** Указываются в README модуля, не обязательно минимизируются, это не главное!
5. **AutoTest:** Каждая библиотека имеет файл автотестов (`ModuleName_AutoTest.au3`)
6. **Инициализация:** Каждая библиотека имеет функцию `_SDK_ModuleName_Init()` для настройки
7. **Логирование:** Все логи пишутся через Utils в `logs/AppName/AppName_ModuleName.log`
   - Формат: `logs/AppName/AppName_Module.log` (например, `logs/MyApp/MyApp_MySQL.log`)
   - Инициализация: `_SDK_Utils_Init("AppName", "ModuleName")` в начале приложения
   - Смена модуля: `$g_sUtils_SDK_ModuleName = "NewModule"` для переключения логов
8. **Тестирование:** Интеграционные тесты в папке `tests/`, шаблоны в `tests/templates/`

### Примеры использования модулей в приложениях:

#### MySQL + Redis (SCADA приложение)
```autoit
#include "..\..\libs\SDK_Init.au3"

; Инициализация SDK
_SDK_Init("SCADA_App", True, 1, 3, True)

; Инициализация Redis для кеширования
_SDK_Redis_Init("127.0.0.1", 6379)

; Чтение данных датчика из MySQL
Local $aData = _MySQL_Select("sensors", "*", "id=1", "", 1, $MYSQL_SERVER_LOCAL, False)

; Кеширование в Redis (быстрый доступ)
If IsArray($aData) Then
    _Redis_Set("sensor:1:temp", $aData[0][2])
    _Redis_Expire("sensor:1:temp", 60) ; TTL 60 секунд
EndIf

; Счетчик деталей (атомарный)
Local $iCount = _Redis_Incr("production:counter")
_Logger_Write("Произведено деталей: " & $iCount, 1)

; Сохранение критических данных на диск
_Redis_Save()
```

#### Кольцевой буфер для логов датчиков
```autoit
#include "..\..\libs\SDK_Init.au3"

_SDK_Init("Sensor_Logger", True, 1, 3, True)
_SDK_Redis_Init("127.0.0.1", 6379)

; Запись данных датчика в кольцевой буфер (последние 100 значений)
Local $fTemp = 25.5
Local $sTimestamp = @YEAR & "-" & @MON & "-" & @MDAY & " " & @HOUR & ":" & @MIN & ":" & @SEC
_Redis_ListPush("sensor:temp:001", $fTemp & "C|" & $sTimestamp, 100)

; Получение последних 10 значений
Local $aHistory = _Redis_ListGetAll("sensor:temp:001")
If IsArray($aHistory) Then
    For $i = 0 To 9
        If $i < UBound($aHistory) Then
            _Logger_Write("История [" & $i & "]: " & $aHistory[$i], 1)
        EndIf
    Next
EndIf
```

### Особенности WebView2:
- **COM интерфейс:** Работает через ObjCreate("NetWebView2.Manager")
- **Событийная модель:** Требует OnEventMode или WM обработчики для COM событий
- **Неблокирующая работа:** Все операции через события (INIT_READY, NAV_COMPLETED, SCRIPT_RESULT)
- **Регистрация:** Требуется regasm для регистрации COM (bin/Register_web2.au3)
- **Исходники:** .NET/C# код в src/ можно изменять только после согласования
- **AutoTest:** 17 тестов покрывают все основные функции (навигация, JS, CSS, cookies, скриншоты)

   - **Приоритет 2:** Прямой запуск .au3 файла (если MCP недоступен)
