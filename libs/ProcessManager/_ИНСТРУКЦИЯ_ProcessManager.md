## 🎯 НАЗНАЧЕНИЕ ПРОЕКТА

**ProcessManager** - это профессиональный менеджер процессов для Windows, написанный на AutoIt. Предназначен для автоматического запуска, мониторинга и управления процессами с расширенными возможностями настройки.

## 🧩 АРХИТЕКТУРА СИСТЕМЫ

### 📁 Модульная структура:
- **Разделение ответственности**: Каждый файл отвечает за свою область
- **Глобальные переменные**: Централизованы в Config
- **Event-driven архитектура**: Использует GUIOnEventMode
- **Система логирования**: Многоуровневая с фильтрацией

### 🧠 ProcessManager_Core.au3 (Основная логика)
**Назначение**: Мониторинг и управление процессами
**Ключевые функции**:
- `InitializeProcessStatuses()` - инициализация статусов при запуске
- `RunStartupProcessesOnce()` - одноразовый запуск процессов
- `MonitorProcesses()` - основной цикл мониторинга
- `RestartProcessLogic()` - логика перезапуска процессов

**Алгоритм работы**:
1. 🔍 Проверка существования процессов каждые 2 секунды
2. ⏰ Обработка таймеров запуска и перезапуска
3. 🔄 Автоматический перезапуск при необходимости
4. 👥 Контроль дубликатов процессов

### 🎮 ProcessManager_Actions.au3 (Действия пользователя)
**Назначение**: Обработка всех действий пользователя
**Основная функция**: `ActionHandler()` - центральный обработчик событий

**Обрабатываемые действия**:
- ✅ Чекбоксы: При запуске, Автозапуск, Дубликаты
- 🔘 Кнопки: Стоп, Плей, Пауза для каждого процесса
- 🎛️ Управляющие кнопки: AUTO ON/OFF, Добавить процесс

**Логика обработки**:
- Поиск процесса по ID элемента управления
- Обновление массива данных
- Сохранение в INI файл
- Логирование действий

### 🛠️ ProcessManager_Utils.au3 (Утилиты)
**Назначение**: Вспомогательные функции
**Основные функции**:
- `_MyErrorHandler()` - обработка критических ошибок
- `_RestartApplication()` - перезапуск приложения
- `TimeStringToSec()` - конвертация времени
- `CalculateWorkTime()` - расчет времени работы

**Система обработки ошибок**:
- Автоматическое логирование ошибок
- Попытки восстановления
- Перезапуск при критических сбоях

### 📝 ProcessManager_Logging.au3 (Логирование)
**Назначение**: Система логирования и отображения
**Основные функции**:

- `ShowLogsNew()` - отображение логов с фильтрацией
- `InitLogFonts()` - инициализация шрифтов для логов

**Типы логов**:
- 🔧 СИСТЕМА - системные события
- 👤 КОНСОЛЬ - действия пользователя  
- ❌ ОШИБКА - ошибки и сбои
- ℹ️ ИНФО - информационные сообщения

**Фильтрация логов**:
- По типу события
- По имени процесса
- Ограничение количества записей (500 макс.)

### 💾 ProcessManager_Settings.au3 (Настройки)
**Назначение**: Загрузка и сохранение настроек
**Основные функции**:
- `LoadSettingsFromINI()` - загрузка из INI
- `SaveSettingsToINI()` - сохранение в INI
- `CreateDefaultINI()` - создание настроек по умолчанию

**Структура данных процесса**:
```
$aProcesses[i][0]  = Имя процесса
$aProcesses[i][1]  = Путь к EXE файлу
$aProcesses[i][2]  = При запуске (0/1)
$aProcesses[i][3]  = Автозапуск (0/1)
$aProcesses[i][4]  = Счетчик перезапусков
$aProcesses[i][5]  = Статус (Runned/Stopped)
$aProcesses[i][6]  = Убивать дубликаты (0/1)
$aProcesses[i][7]  = Режим работы
$aProcesses[i][8]  = Таймер первого запуска
$aProcesses[i][9]  = Таймер перезапуска
$aProcesses[i][10] = Дата последнего запуска
$aProcesses[i][11] = Handle таймера
$aProcesses[i][12] = Флаг первого запуска
$aProcesses[i][13] = Дата последней остановки
```

### 🔧 Как это работает:

#### Система мониторинга:
1. **Adlib функция** вызывается каждые 2000мс
2. **Проверка ProcessExists()** для каждого процесса
3. **Сравнение** с предыдущим состоянием
4. **Обновление GUI** при изменениях
5. **Логирование** всех изменений статуса

#### Автоматический запуск:
1. **Чекбокс "При запуске"** устанавливает флаг
2. **Таймер первого запуска** отсчитывает задержку
3. **Одноразовый запуск** при истечении таймера
4. **Сброс флага** после выполнения

#### Автоматический перезапуск:
1. **Чекбокс "Автозапуск"** активирует режим
2. **Обнаружение остановки** процесса
3. **Проверка таймера перезапуска** 
4. **Запуск процесса** при истечении времени
5. **Увеличение счетчика** перезапусков

#### Контроль дубликатов:
1. **Чекбокс "Дубликаты"** включает контроль
2. **Поиск процессов** с одинаковым именем
3. **Подсчет количества** экземпляров
4. **Завершение лишних** процессов
5. **Логирование действий**
